# 需求文档

## 简介

主页宠物切换箭头位置和交互优化。将箭头从宠物图片下方移到图片左右两侧，支持点击切换，支持循环切换，移除底部进度条。

## 需求

### 需求 1 - 宠物切换箭头重新定位

**用户故事：** 作为宠物主人，我想要在宠物图片左右两侧看到切换箭头，以便更直观地知道可以切换宠物。

#### 验收标准

1. 当用户拥有多只宠物时，宠物图片左侧应显示左箭头（`<`），右侧应显示右箭头（`>`）
2. 当用户只有一只宠物时，不应显示任何箭头和滑动提示
3. 箭头应垂直居中于宠物图片区域

### 需求 2 - 点击箭头切换宠物

**用户故事：** 作为宠物主人，我想要通过点击箭头切换宠物，以便除滑动外有另一种切换方式。

#### 验收标准

1. 当用户点击右箭头时，应切换到下一只宠物
2. 当用户点击左箭头时，应切换到上一只宠物
3. 当显示最后一只宠物时，点击右箭头应循环到第一只宠物
4. 当显示第一只宠物时，点击左箭头应循环到最后一只宠物
5. 原有的滑动切换功能应保持不变

### 需求 3 - 移除底部滑动指示器

**用户故事：** 作为宠物主人，箭头已经足够引导切换，底部进度条不再需要。

#### 验收标准

1. 底部的滑动箭头图片（`arrow-swipe.png`）、滑动提示文案（"左右滑动切换宠物"）和进度条应全部移除
2. 移除后页面布局无异常
