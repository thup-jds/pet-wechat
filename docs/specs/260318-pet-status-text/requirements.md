# 需求文档：宠物状态文字展示

## 简介

在管理后台模拟触发宠物行为事件后，小程序首页宠物形象下方应显示状态文字，让用户能感知到事件已发生。当前系统后端已能记录行为事件（petBehaviors），但前端首页没有展示最近行为的入口。

Closes #22

## 需求

### 需求 1 - 宠物形象下方显示最近行为状态

**用户故事：** 作为宠物主人，我想要在首页宠物形象下方看到宠物的最新行为状态和发生时间，以便确认行为事件已生效并了解宠物当前在做什么。

#### 验收标准

1. 当用户进入首页且宠物有行为记录时，系统应该在宠物形象下方显示最近一次行为的 actionType 和相对时间（如「10分钟前 · walking」）
2. 当用户进入首页且宠物没有任何行为记录时，系统应该在宠物形象下方显示默认文案（如「暂无活动记录」）
3. 当多宠物切换时，系统应该同步切换对应宠物的最近行为状态
4. 当后端行为 API 请求失败时，系统应该静默降级，不影响首页其它功能的展示

### 需求 2 - 后端 API 支持获取最近行为

**用户故事：** 作为前端页面，我需要高效获取每只宠物的最近一条行为记录，以便在首页展示。

#### 验收标准

1. GET `/api/pets` 返回的每只宠物数据中应该包含 `latestBehavior` 字段（包含 actionType 和 timestamp），避免前端为每只宠物单独请求行为 API
2. 当宠物没有行为记录时，`latestBehavior` 应该为 `null`
