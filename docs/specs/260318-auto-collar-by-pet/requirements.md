# 需求文档

## 简介

创建行为事件时，选择宠物后应自动关联该宠物绑定的项圈设备，而非让用户手动选择。涉及管理后台前端、后端 API 校验两个层面。

## 需求

### 需求 1 - 选择宠物后自动过滤项圈候选列表

**用户故事：** 作为管理员，我想要在选择宠物后只看到该宠物绑定的项圈，以便快速创建行为事件而不必在所有项圈中寻找。

#### 验收标准

1. 当用户在管理后台手动创建行为事件时，如果选择了一只宠物，那么项圈下拉列表应该只显示该宠物绑定的项圈设备
2. 当用户在管理后台批量生成行为事件时，如果选择了一只宠物，那么项圈下拉列表应该只显示该宠物绑定的项圈设备
3. 当宠物只绑定了一个项圈时，那么系统应该自动选中该项圈，无需用户手动操作
4. 当宠物绑定了多个项圈时，那么系统应该列出所有绑定的项圈供用户选择，每个项圈需显示上次在线时间和创建时间
5. 当用户切换宠物选择时，那么项圈选择应该重置并按新宠物重新过滤

### 需求 2 - 宠物未绑定项圈时的提示

**用户故事：** 作为管理员，我想要在宠物未绑定项圈时得到明确提示，以便我知道需要先去绑定项圈。

#### 验收标准

1. 当选择的宠物没有任何绑定的项圈设备时，那么系统应该在项圈选择区域显示提示"该宠物未绑定项圈，请先绑定"
2. 当宠物未绑定项圈时，那么创建/生成按钮应该被禁用，阻止提交

### 需求 3 - 后端 API 校验 petId 与 collarDeviceId 的绑定关系

**用户故事：** 作为系统，我需要校验提交的 petId 和 collarDeviceId 确实存在绑定关系，以防止数据不一致。

#### 验收标准

1. 当调用 POST `/api/admin/behaviors` 创建行为事件时，如果提交的 collarDeviceId 对应的项圈的 petId 与提交的 petId 不一致，那么 API 应该返回 400 错误并提示"项圈与宠物不匹配"
2. 当调用 POST `/api/admin/behaviors/auto` 批量生成行为事件时，如果提交的 collarDeviceId 对应的项圈的 petId 与提交的 petId 不一致，那么 API 应该返回 400 错误并提示"项圈与宠物不匹配"
3. 当调用 POST `/api/behaviors`（项圈上报接口）时，如果提交的 collarDeviceId 对应的项圈的 petId 与提交的 petId 不一致，那么 API 应该返回 400 错误并提示"项圈与宠物不匹配"
