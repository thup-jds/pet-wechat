# 需求文档：模拟设备绑定流程

## 简介

当前硬件（项圈、桌面摆台）尚未开发完成，无法进行蓝牙配对等真实设备操作。为了端到端测试小程序的设备绑定流程，需要支持"模拟无主设备"机制：在 Admin 后台创建出厂状态（无主）的设备，小程序通过模拟蓝牙搜索发现这些设备并完成绑定，尽量贴近真实场景。

## 需求

### 需求 1 - 无主设备数据模型

**用户故事：** 作为管理员，我想要创建无主设备（不归属任何用户），以便模拟工厂出货的硬件。

#### 验收标准

1. 当管理员创建项圈或桌面摆台时，如果不选择所属用户，那么系统应该允许创建 userId 为空的设备记录
2. 当设备的 userId 为空时，那么该设备应该被视为"无主设备"（工厂出货状态）
3. 当管理员在设备列表中查看时，那么无主设备应该有明显的标识区分（如标签显示"无主"）
4. 当管理员查看无主设备时，那么应该能方便地复制设备 MAC 地址

### 需求 2 - 项圈模拟搜索与绑定

**用户故事：** 作为小程序用户，我想要通过模拟蓝牙搜索发现后台创建的无主项圈，以便测试完整的项圈绑定流程。

#### 验收标准

1. 当用户在小程序点击"搜索设备"时，那么系统应该调用服务端 API 获取所有无主项圈列表（替代硬编码的 mock 设备）
2. 当搜索到无主项圈时，那么列表中每个设备旁应显示红色"Mock"标识
3. 当用户从列表中选择一个项圈时，那么系统应该进入 WiFi 配网页面（显示模拟配网成功）
4. 当配网流程完成时，那么系统应该将该项圈的 userId 设为当前用户，完成认领绑定
5. 当项圈被认领后，那么该项圈不应再出现在其他用户的搜索结果中

### 需求 3 - 桌面摆台模拟搜索与绑定

**用户故事：** 作为小程序用户，我想要通过模拟蓝牙搜索发现后台创建的无主桌面摆台，以便测试完整的摆台绑定流程。

#### 验收标准

1. 当用户在小程序点击搜索桌面摆台时，那么系统应该调用服务端 API 获取所有无主桌面摆台列表
2. 当搜索到无主摆台时，那么列表中每个设备旁应显示红色"Mock"标识
3. 当用户从列表中选择一个摆台时，那么系统应该走配网流程（模拟成功）
4. 当配网完成时，那么系统应该将该摆台的 userId 设为当前用户，完成认领
5. 当摆台被认领后，那么用户应该进入宠物配对步骤（选择宠物绑定到摆台）

### 需求 4 - Admin 后台设备管理优化

**用户故事：** 作为管理员，我想要在后台方便地管理模拟设备的生命周期，以便高效地进行测试。

#### 验收标准

1. 当管理员创建设备时，那么 userId 字段应该为可选（默认不选 = 无主设备）
2. 当管理员查看设备列表时，那么应该能区分"无主"和"已绑定"设备
3. 当管理员查看已绑定设备时，那么应该显示绑定用户的信息
