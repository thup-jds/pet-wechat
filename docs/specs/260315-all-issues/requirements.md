# 需求文档

## 简介

批量修复 YEHEY 宠物小程序 MVP 的 7 个 GitHub Issue（#3~#9），涵盖分享机制统一、安全区适配、页面布局修复、图片资源接入、真实文件存储、多图上传、UI 审查优化。

## 需求

### 需求 1 - 统一宠物分享机制（Issue #3）

**用户故事：** 作为宠物主人，我想要通过微信卡片分享宠物给好友或群，以便好友可以接受邀请查看我的宠物。

#### 验收标准

1. 当用户在设备页点击"分享授权"时，系统应生成 invite code 并调用 `Taro.shareAppMessage` 发送微信小程序卡片
2. 当好友点击卡片打开小程序时，系统应显示邀请预览页（宠物名、分享者昵称）
3. 当好友接受邀请时，系统应创建 desktopPetBindings（authorized 类型），将宠物绑到接受者的所有摆台
4. 当同一用户重复接受同一邀请时，系统应提示"已接受过此邀请"而不重复绑定
5. 系统应移除所有 share-links 相关代码：`shareLinks`、`shareRecords` 表、相关路由、前端引用
6. 系统应移除 shared types 中的 `ShareLink`、`ShareRecord`、`ShareType`、`ShareLinkStatus` 类型

### 需求 2 - iPhone 刘海屏安全区适配（Issue #4）

**用户故事：** 作为 iPhone 用户，我希望页面内容不被状态栏和刘海遮挡，以便正常使用小程序。

#### 验收标准

1. 所有自定义页面顶部应正确添加安全区 padding（使用 `statusBarHeight`）
2. 使用 NavBar 组件的页面已通过 NavBar 内部处理（已实现 ✅）
3. 主页（tab 页无 NavBar）应在 top-bar 中使用 `statusBarHeight` 作为 padding-top
4. 设备页的自定义导航栏应添加安全区适配

### 需求 3 - 引导页布局修复（Issue #5）

**用户故事：** 作为首次使用的用户，我希望引导页在一屏内完整展示，以便快速了解功能。

#### 验收标准

1. "狗头 + 链接符号 + 项圈/水晶球"应水平排列，而非纵向堆叠
2. 图标尺寸应缩小，确保两个引导卡片 + "跳过"按钮在一屏内完整展示
3. 使用设计稿中的图片资源替换 SVG 占位符
4. 整体布局紧凑，留白合理

### 需求 4 - 主页设计稿对齐（Issue #6）

**用户故事：** 作为用户，我希望主页展示真实的宠物图片和完整的功能按钮，以获得良好的使用体验。

#### 验收标准

1. 中央区域应展示 927×927 宠物大图（使用 `image-import-31.png`）
2. 左上角头像应使用真实宠物图片（使用 `image-import-52.png`）
3. 底部应有 3 个圆形功能按钮（用户 `image-import-16.png`、数据 `image-import-41.png`、设置 `image-import-14.png`）
4. 对话气泡应使用图片资源 `image-import-40.png` 作为背景
5. 滑动指示器应包含箭头图片 + "左右滑动切换宠物" 文字 + 进度条
6. 通知铃铛应移至左侧活跃值下方，带红点提示
7. 设备卡片区域应使用图片 icon（`image-import-5.png` 项圈、`image-import-11.png` 桌面端）

### 需求 5 - 真实文件存储（Issue #7）

**用户故事：** 作为用户，我上传的宠物图片应被真正存储并可访问，以便后续的宠物形象定制流程正常工作。

#### 验收标准

1. 后端应使用 MinIO（S3 兼容）作为对象存储，通过 Docker Compose 部署
2. 上传后返回真实可访问的图片 URL
3. 添加文件类型校验（仅允许 jpg/png/webp）
4. 添加文件大小限制（10MB）
5. 前端 BASE_URL 从 request.ts 的统一配置获取，移除 pet-avatar 中的硬编码

### 需求 6 - 多图上传修复（Issue #8）

**用户故事：** 作为用户，我选择的所有宠物图片都应被上传，而非仅上传第一张。

#### 验收标准

1. 前端应循环上传所有用户选中的图片
2. 上传结果的 URL 列表应传递给后续的定制任务接口
3. 上传过程应显示进度（如"上传中 2/5"）

### 需求 7 - UI 审查与优化（Issue #9）

**用户故事：** 作为用户，我希望小程序界面在交互一致性、视觉层次等方面有统一规范。

#### 验收标准

1. 按钮样式统一：主要按钮、次要按钮风格一致
2. 色彩对比度满足可访问性要求
3. 触控区域尺寸不小于 44×44px（rpx 等效）
4. 加载状态、错误提示风格统一
5. 页面切换和状态变化有合理过渡
6. 字体层级、间距节奏、留白运用合理
