# 需求文档

## 简介

修复项圈绑定（collar-bind）和桌面端绑定（desktop-bind）页面中，模拟蓝牙搜索无结果后按钮无法再次点击的 bug。对应 GitHub Issue #19。

## 需求

### 需求 1 - 搜索无结果后可重新搜索

**用户故事：** 作为宠物主人，我想要在模拟蓝牙搜索无结果后能重新搜索设备，以便在管理后台创建模拟设备后再次尝试绑定。

#### 验收标准

1. 当用户在项圈绑定页面点击「Mock 搜索设备」且 API 返回空列表时，那么页面应显示"未发现可用设备"提示，且按钮文案恢复为「Mock 搜索设备」，用户可再次点击触发搜索
2. 当用户在桌面端绑定页面点击「Mock 搜索设备」且 API 返回空列表时，那么页面应显示"未发现可用设备"提示，且按钮文案恢复为「Mock 搜索设备」，用户可再次点击触发搜索
3. 当用户重新搜索且 API 返回非空列表时，那么页面应正常显示设备列表，按钮变为「Mock 连接设备」，用户可选择设备并连接
